Projekt polega na prostym opracowaniu statystycznym wynikw porwnania dziaania wybranych algorytmw minimalizacji stochastycznej. Zaimplementowalimy algorytmy PRS oraz GA. Do porwnania wybralimy funkcje Alpine01 i Schwefel.

Przedstawienie funkcji

Alpine01

Funkcja Alpine01 jest funkcj testow wykorzystywan w optymalizacji, szczeglnie w badaniach nad algorytmami optymalizacji globalnej. Charakteryzuje si wieloma lokalnymi minimami, co sprawia, e jest trudna do optymalizacji. Funkcja jest zdefiniowana jako:

\[ f(x) = \sum_{i=1}^{d} \left( |x_i \sin(x_i) + 0.1 x_i| \right) \]

gdzie:
- \(x_i\) to elementy wektora zmiennych wejciowych,
- \(d\) to liczba wymiarw funkcji (liczba zmiennych wejciowych),
- funkcja jest zwykle testowana w zakresie \(x_i \in [-10, 10]\).

Cechy funkcji:

  • Globalne minimum: Funkcja ma globalne minimum w punkcie \(x = 0\) (dla wszystkich zmiennych \(x_i = 0\)).
  • Lokalne minima: Funkcja ma wiele lokalnych minimw, co czyni j wymagajc w testowaniu algorytmw optymalizacji.
  • Nieliniowo: Funkcja jest nieliniowa, co sprawia, e jej analiza i optymalizacja stanowi wyzwanie.

Funkcja Schwefela

Funkcja Schwefela jest jedn z klasycznych funkcji testowych wykorzystywanych w optymalizacji. Ma wiele lokalnych minimw, co czyni j trudnym przypadkiem do rozwizania przy uyciu algorytmw optymalizacji. Funkcja Schwefela jest zdefiniowana jako:

\[ f(x) = 418.9829 \cdot d - \sum_{i=1}^{d} x_i \sin\left(\sqrt{|x_i|}\right) \]

gdzie:
- \(x_i\) to elementy wektora zmiennych wejciowych,
- \(d\) to liczba wymiarw funkcji,
- funkcja jest zwykle testowana w zakresie \(x_i \in [-500, 500]\).

Cechy funkcji:

  • Globalne minimum: Funkcja osiga swoje globalne minimum w punkcie \(x = 420.9687\) (gdzie \(x_i = 420.9687\) dla wszystkich \(i\)).
  • Lokalne minima: Funkcja ma wiele lokalnych minimw, co czyni j wyzwaniem dla algorytmw optymalizacji globalnej.
  • Nieliniowo i trudno: Funkcja jest nieliniowa i wykazuje skomplikowan struktur, pen zagi i oscylacji, przez co jest trudna do optymalizacji.

# 1. Pobieranie prawdziwych dziedzin ---------------------------------------------------------
get_domain <- function(fn) {
ps <- getParamSet(fn)
list(
lower = sapply(ps$pars, function(p) p$lower),
upper = sapply(ps$pars, function(p) p$upper)
)
}

# 2. Poprawiona funkcja generujca punkty ----------------------------------------------------
getRandomPoint <- function(dimensions, domain) {
sapply(1:dimensions, function(i) {
runif(1, domain$lower[i], domain$upper[i])
})
}

Implementujemy algorytmy PRS i GA:

# 3. Uniwersalna funkcja PRS -----------------------------------------------------------------
performPRS <- function(n_eval, fn, domain) {
  dimensions <- length(domain$lower)
  points <- matrix(nrow = n_eval, ncol = dimensions)
  for(i in 1:n_eval) {
    points[i,] <- getRandomPoint(dimensions, domain)
  }
  min(apply(points, 1, fn))
}

# 4. Uniwersalna funkcja GA ------------------------------------------------------------------
performGA <- function(n_runs, n_eval, fn, domain) {
  dimensions <- length(domain$lower)
  res <- replicate(n_runs, ecr(
    fitness.fun = fn,
    n.dim = dimensions,
    lower = domain$lower,
    upper = domain$upper,
    minimize = TRUE,
    representation = "float",
    mu = 50L,
    lambda = 25L,
    terminators = list(stopOnEvals(n_eval)),
    mutator = setup(mutGauss, lower = domain$lower, upper = domain$upper)
  )$best.y)
  res
}

Tworzymy funkcj przeprowadzajc badania:

run_experiment <- function(fn_name, dimensions, n_runs = 100) {
  # Inicjalizacja funkcji
  fn <- switch(fn_name,
               "Alpine01" = makeAlpine01Function(dimensions),
               "Schwefel" = makeSchwefelFunction(dimensions))

  domain <- get_domain(fn)
  true_min <- ifelse(fn_name == "Alpine01", 0, -418.9829 * dimensions)

  # PRS
  prs_results <- replicate(n_runs, performPRS(1000, fn, domain))

  # GA
  ga_results <- performGA(n_runs, 1000, fn, domain)

  list(
    prs = prs_results,
    ga = ga_results,
    stats = data.frame(
      Function = fn_name,
      Dimensions = dimensions,
      PRS_mean = mean(prs_results),
      GA_mean = mean(ga_results),
      True_min = true_min
    )
  )
}

visualize_histogram <- function(data, true_min) {
  ggplot(data, aes(x = value, fill = algorithm)) +
    geom_histogram(position = "identity", alpha = 0.7, bins = 30) +
    geom_vline(xintercept = true_min, color = "red", linetype = "dashed") +
    facet_wrap(~algorithm, ncol = 1) +
    labs(title = paste("Histogram wynikw dla", unique(data$function_dim)),
         x = "Warto funkcji", y = "Czstotliwo") +
    theme_bw()
}

visualize_violin <- function(data, true_min) {
  ggplot(data, aes(x = algorithm, y = value, fill = algorithm)) +
    geom_violin(trim = FALSE, show.legend = FALSE) +
    geom_boxplot(width = 0.1, fill = "white", outlier.size = 1) +
    geom_hline(yintercept = true_min, color = "red", linetype = "dashed") +
    labs(title = paste("Wykres skrzypcowy dla", unique(data$function_dim)),
         x = "Algorytm", y = "Warto funkcji") +
    theme_bw() +
    theme(legend.position = "none")
}

1. Funkcja Alpine01

Funkcja Alpine01 wykazuje szereg lokalnych minimw, co czyni j wymagajc dla algorytmw optymalizacyjnych. Analiza wynikw dla rnych wymiarw tej funkcji (2, 10, 20) pokazuje, e oba algorytmy (PRS i GA) radziy sobie stosunkowo dobrze w porwnaniu do funkcji Schwefela, uzyskujc wyniki bliskie globalnemu minimum.

  • Dla wymiaru 2:

  • PRS osign warto redni 0.065, a GA uzyska wynik 0.001. Oba algorytmy uzyskay bardzo zbliony wynik do True_min = 0, co sugeruje, e algorytmy poradziy sobie bardzo dobrze z optymalizacj w tym przypadku.

  • Rnica wynosi 0.064 (PRS) i 0.001 (GA), co pokazuje, e GA uzyska wynik bliszy rzeczywistemu minimum.

  • Dla wymiaru 10:

  • PRS uzyska wynik 9.716, a GA wynik 2.554, przy czym True_min wynosi 0.

  • Rnice wynosz 7.162 dla PRS i 2.554 dla GA, co pokazuje, e GA wykaza si lepszym wynikiem w porwnaniu do PRS. Cho obydwa algorytmy miay trudnoci w osigniciu dokadnego minimum, GA okaza si bardziej skuteczny w tym przypadku.

  • Dla wymiaru 20:

  • PRS uzyska wynik 29.693, a GA wynik 8.732.

  • Rnice wynosz 20.961 (PRS) i 8.732 (GA), a mimo to wyniki obu algorytmw s wci znacznie lepsze w porwnaniu do funkcji Schwefela, cho nadal znaczco odbiegaj od globalnego minimum. GA wykazuje lepsze wyniki w tym przypadku.

2. Funkcja Schwefel

Funkcja Schwefela jest bardziej zoona i posiada wiksz liczb lokalnych minimw, co stanowi wiksze wyzwanie dla algorytmw optymalizacyjnych. Wyniki dla rnych wymiarw funkcji Schwefela pokazuj, e oba algorytmy maj wiksze trudnoci w uzyskaniu wartoci bliskich globalnemu minimum.

  • Dla wymiaru 2:

  • PRS uzyska wynik -802.956, a GA wynik -654.641, podczas gdy True_min wynosi -837.966.

  • Rnice wynosz 148.315 dla PRS i 183.325 dla GA, co pokazuje, e oba algorytmy utkny w lokalnych minimach i nie byy w stanie znale wartoci bliskiej globalnemu minimum.

  • Dla wymiaru 10:

  • PRS osign wynik -1996.004, a GA wynik -1476.651, przy czym True_min wynosi -4189.829.

  • Rnice wynosz 519.353 dla PRS i 1713.178 dla GA, co wskazuje, e GA mia znacznie wiksz trudno w optymalizacji funkcji Schwefela i bardziej oddali si od rzeczywistego minimum w porwnaniu do PRS.

  • Dla wymiaru 20:

  • PRS uzyska wynik -2772.805, a GA wynik -2063.234, a True_min wynosi -8379.658.

  • Rnice wynosz 709.571 dla PRS i 5736.424 dla GA. Wyniki s znacznie gorsze w przypadku funkcji Schwefela, a szczeglnie dla GA, ktry mia trudnoci z optymalizowaniem tej funkcji w wyszych wymiarach.
